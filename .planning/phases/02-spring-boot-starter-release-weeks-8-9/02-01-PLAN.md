<objective>
Lay the foundation for the Spring Boot starter: (1) enhance the core BudgetCircuitBreaker so live in-flight agents are visible and aggregate breach counts are queryable (D-08), (2) add the model name to CallTracked events so downstream metrics can tag {agent, model} (D-09/D-11, Claude's-discretion Option A), and (3) wire up the starter module's build.gradle.kts with Spring Boot + Micrometer dependencies.

Purpose: Every observability component in this phase (actuator endpoint, health indicator, metrics collector) reads from the breaker's snapshot/aggregate APIs and from CallTracked.model. This plan creates those contracts first so downstream plans build against a stable surface, not a scavenger hunt. The starter build must compile before any starter source lands.

Output: Modified core (additive, no API breaks) + the starter build file. Core tests extended to cover the new snapshot and aggregate APIs.
</objective>
